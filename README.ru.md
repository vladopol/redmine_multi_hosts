# Redmine Multi Hosts

[English version](README.md)

Плагин для Redmine, позволяющий запускать одну установку Redmine с несколькими доменными именами, каждое из которых может иметь собственный брендинг, адрес отправителя писем и настройки SMTP.

Это форк [florianeck/redmine_multi_hosts](https://github.com/florianeck/redmine_multi_hosts) с исправлениями совместимости для Redmine 4.x / Rails 5.x и новыми возможностями.

## Что нового в этом форке

- **Совместимость с Rails 5.x / Redmine 4.x** — исправлены устаревшие `before_filter`, `after_filter`, `update_attributes` и синтаксис миграций
- **Настройки SMTP для каждого хоста** — каждый хост может использовать свой SMTP-сервер, учётные данные и адрес отправителя
- **Привязка проекта к организации** — можно закрепить проект за конкретным хостом, чтобы его уведомления всегда отправлялись от имени этой организации, независимо от того, через какой домен вошёл пользователь
- **Брендинг заголовка страницы** — название приложения теперь корректно подменяется на всех страницах, включая страницы проектов
- **Интерфейс создания хоста** — добавлена кнопка "Новый хост" и форма прямо в админ-панели
- **Локализация** — английский и русский переводы

## Требования

- Redmine 4.x
- Rails 5.2.x

## Установка

```bash
cd /path/to/redmine/plugins
git clone https://github.com/vladopol/redmine_multi_hosts.git
cd /path/to/redmine
bundle exec rake redmine:plugins:migrate RAILS_ENV=production
touch tmp/restart.txt
```

## Настройка

После установки зайдите в **Администрирование → MultiHost** и:

1. Нажмите **Новый хост**, чтобы добавить основной хост (или используйте rake-задачу ниже)
2. Добавьте дополнительные хосты со своим брендингом и настройками SMTP

Либо используйте rake-задачи:

```bash
# Создать хост по умолчанию на основе текущих настроек Redmine
bundle exec rake multi_hosts:setup_default_host RAILS_ENV=production

# Добавить новый хост
bundle exec rake "multi_hosts:add_host[https://support.example.com/]" RAILS_ENV=production
```

## Настройки хоста

Для каждого хоста доступны:

- **Название приложения** — отображается в заголовке страницы и в письмах
- **Email отправителя** — адрес, с которого отправляются уведомления
- **Настройки SMTP** — адрес сервера, порт, пользователь, пароль, метод аутентификации

## Привязка проекта к организации

По умолчанию письмо-уведомление отправляется через тот хост, под которым в данный момент авторизован пользователь, инициировавший уведомление — это исходное поведение плагина. Если ваш Redmine обслуживает несколько организаций и один и тот же пользователь (или администратор) может иногда заходить через один домен, работая при этом над проектом, принадлежащим другой организации, это может привести к отправке уведомлений не от той организации.

Чтобы этого избежать, откройте настройки проекта и установите поле **Организация (MultiHost)** в значение хоста, которому принадлежит проект. С этого момента все уведомления по этому проекту (задачи, новости, документы, сообщения, страницы wiki) всегда будут использовать брендинг, адрес отправителя и настройки SMTP этого хоста, независимо от того, через какой домен вошёл пользователь, инициировавший уведомление.

Оставьте поле пустым (**отсутствует**), чтобы сохранить для этого проекта прежнее поведение — по домену входа.

Хост по умолчанию тоже доступен в этом списке, так что проект можно явно закрепить и за ним, вместо того чтобы полагаться на домен входа.

## Как это работает

Для каждого исходящего письма плагин определяет, чьи настройки применить, в следующем порядке:

1. **Привязка проекта** — если у проекта, к которому относится уведомление (определяется по заголовку `X-Redmine-Project`, который ядро Redmine само добавляет в письма), явно указана **Организация (MultiHost)**, используется этот хост.
2. **Домен входа** — иначе используется хост, соответствующий домену, через который в данный момент авторизован инициировавший уведомление пользователь.
3. **Хост получателя** — как запасной вариант, используется хост получателя письма, если он привязан не к хосту по умолчанию.

После того как хост определён, плагин:
- подменяет заголовок страницы и заголовок вкладки браузера;
- использует настройки SMTP этого хоста для доставки письма;
- подменяет адрес отправителя и название приложения в теме и тексте письма.
